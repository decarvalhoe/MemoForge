# Audit d'accessibilité (cible WCAG 2.2 AA)

> Deux garde-fous automatisés en CI : le **contraste des tokens** (`tests/a11y/contrast.test.mjs`,
> ci-dessous) et **axe-core** sur les vues rendues ([section E10-4](#audit-runtime-axe-core--sémantique--e10-4-165)).

Audit **exécutable** : les ratios ci-dessous sont calculés par `src/util/contrast.js` et
**gardés par `tests/a11y/contrast.test.mjs`** — toute régression d'un token sous son seuil
fait échouer la CI.

Seuils AA : **4.5:1** pour le texte normal, **3:1** pour le grand texte (≥ ~18px bold /
24px) et les composants d'interface (bordures, glyphes, états).

### Trouvailles & contraintes d'usage (thème clair)

1. **`--text-muted` (4.34:1)** est **sous** le seuil texte normal (4.5). → **À réserver au
   grand texte ou aux libellés d'UI** (kickers, meta ≥ 18px bold), jamais au petit corps de
   texte. Le test l'impose explicitement.
2. **`--pointer` (3.40:1)** de même : usage **grand texte / élément d'UI** uniquement
   (bordures de casier pointeur, glyphes, badges) — ce qui est son emploi réel dans le design.

### Recommandations (corrections possibles, hors périmètre de cette PR)

Pour rendre ces deux tokens conformes au **texte normal** en thème clair sans changer
l'identité, assombrir légèrement :
- `--text-muted` clair : `#4E7A5C` → viser ≈ `#43704F` (≈ 4.5:1).
- `--pointer` clair : `#B5731A` → viser ≈ `#8A5A12` (≈ 4.5:1) si on veut l'utiliser en petit texte.

Ces ajustements touchent `styles/tokens/colors.css` (design system) → à traiter dans une PR
de tokens dédiée, avec re-validation du styleguide.

## Audit runtime (axe-core) & sémantique — E10-4 (#165)

Au-delà du contraste des **tokens** (gardé ci-dessus), le harnais visuel
(`npm run test:visual`) exécute **axe-core** sur trois vues rendues — carte FR, carte EN et
une salle jouable — et **échoue la CI** sur toute violation `serious`/`critical` (tags
`wcag2a`/`wcag2aa`/`wcag21aa`/`wcag22aa`). État : **zéro violation** (best-practice inclus).

Corrections livrées :

| Sujet | Correction |
|---|---|
| Contraste (1.4.3) | `.mf-locker__addr` : `opacity:.7` retirée (cumulée à `--text-muted`, elle passait sous 4.5:1 — détecté par axe) |
| Focus visible (2.4.7) | anneau `--ring` sur **tous** les interactifs (`button`, `a`, `[role=button]`, `[tabindex]`) |
| Contournement (2.4.1) | lien d'évitement « Aller au contenu / Skip to content » (masqué hors focus, bilingue) |
| Hiérarchie de titres (1.3.1) | `LA RAM`, titres de mission et de région passés en `<h2>`/`<h3>` réels (h1→h2→h3), rendu pixel-identique (harnais à 0 %) |
| Préférences (1.4.12) | ajout de `prefers-contrast: more` (bordures/texte renforcés, scanlines coupées) |

**Couverture manuelle** : clavier complet (poser via palette, **réordonner** ▲/▼ avec
`aria-label`, retirer ×, entrer dans une salle `Enter`/`Espace`) — le glisser-déposer souris
est un complément (équivalence WCAG 2.1.1) ; verdict en `role="status"`/`aria-live="polite"` ;
`<html lang>` suivant la bascule FR/EN ; `prefers-reduced-motion` (exécution instantanée).

**Limites** : réordonnancement souris non-clavier (alternative ▲/▼ couvre le besoin) ;
l'audit axe porte sur 3 vues représentatives partageant les composants des vues secondaires.
